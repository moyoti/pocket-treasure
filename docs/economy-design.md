# 寻宝记经济系统架构设计文档

## 一、系统概述

### 1.1 设计目标
为寻宝记游戏设计一套完整的经济系统，实现：
- 金币作为主要流通货币
- 多元化的金币获取与消耗渠道
- 防止通胀的货币平衡机制
- 玩家间交易市场

### 1.2 核心模块

| 模块 | 功能 | 金币流向 |
|------|------|----------|
| NPC商店 | 出售宝藏给NPC | 产出 |
| 宝箱系统 | 开启高级宝箱 | 消耗 |
| 抽奖系统 | 幸运转盘抽奖 | 消耗 |
| 市场系统 | 玩家间交易 | 手续费回收 |
| 强化系统 | 宝藏升级 | 消耗 |

## 二、价格体系

### 2.1 稀有度基础价格

| 稀有度 | NPC收购价 | 市场参考价 |
|--------|-----------|------------|
| common (普通) | 5 金币 | 10-50 金币 |
| rare (稀有) | 25 金币 | 50-250 金币 |
| epic (史诗) | 100 金币 | 200-1000 金币 |
| legendary (传说) | 500 金币 | 1000-5000 金币 |

### 2.2 宝箱开启费用

| 宝箱类型 | 开启费用 | 掉落池 |
|----------|----------|--------|
| 木箱 | 免费 | 普通物品 |
| 铁箱 | 100 金币 | 普通+稀有 |
| 金箱 | 500 金币 | 稀有+史诗 |
| 传说箱 | 2000 金币 | 史诗+传说 |

### 2.3 抽奖价格

- 单抽: 100 金币
- 十连: 900 金币 (九折)
- 保底: 10抽必出稀有+

## 三、货币平衡机制

### 3.1 金币产出

| 来源 | 日均产出 | 占比 |
|------|----------|------|
| 出售宝藏给NPC | 500 | 55% |
| 每日任务 | 300 | 33% |
| 登录奖励 | 100 | 11% |

### 3.2 金币消耗

| 去向 | 日均消耗 | 占比 |
|------|----------|------|
| 开启宝箱 | 200 | 33% |
| 抽奖 | 100 | 17% |
| 强化 | 200 | 33% |
| 市场手续费 | 100 | 17% |

### 3.3 防通胀机制

1. **市场手续费 (10%)**: 每笔交易回收金币
2. **每日获取上限**: 出售上限5000金币
3. **价格保护**: 市场价格范围限制
4. **强化失败**: 金币消耗不返还

## 四、API接口

### 4.1 经济模块
- `GET /api/economy/balance` - 获取金币余额
- `GET /api/economy/transactions` - 获取交易记录
- `POST /api/economy/sell` - 出售物品给NPC

### 4.2 商店模块
- `GET /api/shop/items` - 获取商店物品
- `POST /api/shop/purchase` - 购买物品

### 4.3 宝箱模块
- `GET /api/chests` - 获取宝箱列表
- `POST /api/chests/open` - 开启宝箱

### 4.4 抽奖模块
- `GET /api/gacha/pools` - 获取抽奖池
- `POST /api/gacha/pull` - 进行抽奖

### 4.5 市场模块
- `GET /api/market/listings` - 获取市场挂单
- `POST /api/market/list` - 创建挂单
- `POST /api/market/buy/:id` - 购买
- `POST /api/market/cancel/:id` - 取消挂单

### 4.6 强化模块
- `POST /api/upgrade` - 强化物品

## 五、数据库实体

### 新增实体

1. **CoinTransaction** - 金币交易记录
2. **ShopItem** - 商店物品
3. **ChestDefinition** - 宝箱定义
4. **UserChest** - 用户宝箱
5. **GachaPool** - 抽奖池
6. **GachaRecord** - 抽奖记录
7. **MarketListing** - 市场挂单
8. **ItemUpgrade** - 强化记录
9. **ItemPrice** - 物品价格配置

### 修改实体

1. **User** - 添加 totalCoinsEarned, totalCoinsSpent
2. **Item** - 添加 basePrice, canSell, canTrade
3. **InventoryItem** - 添加 upgradeLevel, isLocked
